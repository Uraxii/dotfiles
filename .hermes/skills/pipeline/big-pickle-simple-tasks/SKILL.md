# big-pickle-simple-tasks

Big problem → small concrete steps. 15-30min each.

## Method

1. **Assess** whole scope + goal.
2. **First step**: smallest forward momentum. 15-30min.
3. **Chain**: each step unlocks next. Verb-first. Completion criteria. <2hr. Dependencies + risks flagged.
4. **Prioritize**: if too long → MVP path first.

## Output per task

- **Task**: verb-first action
- **Why**: advances goal
- **Done when**: concrete criteria
- **Time**: realistic estimate
- **Next decision**: what to eval before continue

## Rules

- No vague ("plan more", "think about X") → observable actions only.
- Flag tasks needing external input.
- Highlight risk-reducers + assumption-validators early.
- >4hr task → break down further.
- Quick win option for momentum.
- High uncertainty → experiment w/ timebox.
- >12 tasks in phase → group into milestones.

## Stance

Ambiguous goal → ask but always propose concrete starting step. "Here's first step we can refine."
