# Shell

Zsh + starship prompt + ghostty terminal + tmux multiplexer.

## .zshrc

### Purpose

Interactive Zsh setup. Initialises starship, key bindings, aliases,
and tool integrations (nvm, zoxide, opencode).

### Key files

- `zsh/.zshrc` — main config (deployed to `~/.config/zsh/.zshrc`).
- `zsh/.zprofile` — login env (currently exports `QT_QPA_PLATFORMTHEME=qt6ct`).
- `~/.zshenv` — one-line stub, NOT tracked: `export ZDOTDIR="${XDG_CONFIG_HOME:-$HOME/.config}/zsh"`.

### Keybindings & UX

| Sequence | Bind | Action |
|----------|------|--------|
| `^[[H` | Home | beginning-of-line |
| `^[[F` | End | end-of-line |
| `^[[3~` | Del | delete-char |
| `^[[1;5C` | Ctrl+Right | forward-word |
| `^[[1;5D` | Ctrl+Left | backward-word |

Alias: `logout` -> `swaymsg exit` (only when `$SWAYSOCK` set).

Path: prepends `$HOME/.local/bin` and `/home/nikki/.opencode/bin`.

### Theming integration

Sources starship from runtime `~/.config/starship.toml` (generated by
`set-theme.sh` from `.config/starship.toml.tmpl` + the active theme's
`data/starship-palette`). See [theming.md](theming.md).

### External dependencies

`zsh`, `starship`, `zoxide`, `nvm` (optional), `opencode` (optional).

## starship

### Purpose

Cross-shell prompt. Minimal left side (`$directory$character`), full
module set on the right via `right_format = "$all"`. Battery percentage
and local date/time live in the right prompt by default. Theme-driven
palette selection.

### Key files

- `.config/starship.toml.tmpl` — live template (tracked in repo, base
  derived from omerxx/dotfiles starship.toml + battery + time modules
  + per-sway-theme palette blocks).
- `.config/starship.toml` — runtime, generated by `set-theme.sh` (NOT
  tracked).
- Placeholder syntax: `##PALETTE##` (TOML-safe).

### Theming integration

`set-theme.sh` sources `themes/<name>/data/starship-palette`
(shell-style `STARSHIP_PALETTE="<name>"`) and substitutes the single
`##PALETTE##` placeholder. The named palette must exist as a
`[palettes.<name>]` block inside `starship.toml.tmpl`. Currently
defined: `gruvbox`, `synthwave-84`, `catppuccin_mocha`.

To add a palette for a new theme:
1. Add a `[palettes.<new>]` block in `.config/starship.toml.tmpl`.
2. Add `.config/sway/themes/<new>/data/starship-palette` containing
   `STARSHIP_PALETTE="<new>"`.

### External dependencies

`starship`.

## ghostty

### Purpose

GPU-accelerated terminal emulator. Set as `$term` in sway.

### Key files

- `.config/ghostty/config`.

Active settings: `theme = "Kanagawa Dragon"`,
`background-opacity = 0.8`, `font-family = 0xProto Nerd Font Mono`,
`font-size = 14`. Sway also applies a per-window opacity rule (see
`sway/modules/windows`).

### Theming integration

Currently independent of the sway theming pipeline (ghostty has its
own theme key). Add a templated theme key here if/when ghostty enters
the pipeline.

### External dependencies

`ghostty`, `0xProto Nerd Font` (id `nerd-fonts-0xproto` in `deps.toml`).

## tmux

### Purpose

Terminal multiplexer. Provides persistent top status bar (git branch,
full pwd, battery, date/time) that the inline starship prompt cannot —
status bar stays pinned regardless of scroll. Also gives session
persistence, pane splits, and resurrect-on-reboot.

### Key files

- `.config/tmux/tmux.conf` — main config. Prefix `C-a`. Status pinned
  top, refresh every 5s.
- `.config/tmux/tmux.reset.conf` — keybinds (vi-style pane nav, splits,
  copy mode). Sourced from `tmux.conf`.
- `.zshrc` — autostarts tmux on shell launch via
  `exec tmux new-session -A -s root` (attach if `root` session exists,
  else create). Guarded by `$TMUX` (skip if already inside) and
  `command -v tmux` (skip if not installed).

### Keybindings & UX

Prefix: `C-a` (Ctrl+a). Press, release, then the bind.

| Bind | Action |
|------|--------|
| `\|` / `v` | Split horizontal (new pane right) |
| `-` / `s` | Split vertical (new pane below) |
| `h j k l` | Move between panes (vi-style) |
| `, . = +` | Resize pane L/R/U/D (repeatable) |
| `z` | Toggle pane zoom |
| `x` | Kill pane |
| `C-a` | Last window |
| `H` / `L` | Prev / next window |
| `S` | Choose session |
| `R` | Reload `tmux.conf` |
| `r` | Rename window |
| `K` | Send clear+Enter to pane |
| `[` (default) → `v` then `y` | Copy-mode select + yank to Wayland clipboard (`wl-copy`) |

Status-right format: ` <branch> │ <full-pwd> │ <battery>% │ YYYY-MM-DD HH:MM`.
Branch silently empty outside git repos; battery silently empty on
desktops without `/sys/class/power_supply/BAT*`.

### Theming integration

Wired into the sway theming pipeline. `tmux.conf` ships safe-default
styles (named colors) and ends with
`source-file -q ~/.local/share/tmux/theme.conf`.

`set-theme.sh` copies the active theme's
`.config/sway/themes/<name>/data/tmux-theme.conf` to that path and
`tmux source-file`s it into any running server — colors swap live, no
restart. Generated path lives under `~/.local/share/` (not `~/.config/`)
to avoid stow tree-folding writing back into the repo.

Each theme's `tmux-theme.conf` re-sets `status-style`, `status-left`,
`status-right`, `window-status-{,current-}style`, `pane-{,active-}border-style`,
`message-style`, `mode-style` with literal hex from the theme palette
(no placeholder substitution — tmux doesn't expand user options inside
style strings).

### Plugin manager (optional)

`tmux.conf` declares `@plugin` entries for tpm + tmux-sensible +
tmux-yank + tmux-resurrect + tmux-continuum. They no-op until tpm is
cloned:

```bash
git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
# Then inside tmux: <prefix> I  (capital i) to install plugins.
```

Resurrect saves layout to `~/.local/share/tmux/resurrect/`; continuum
auto-saves every 15min.

### External dependencies

`tmux` (in `deps.toml`). Optional: `git` clone of tpm. `wl-copy` from
`wl-clipboard` for copy-mode yank (also in `deps.toml`).
